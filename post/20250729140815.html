RSA相关总结

发表于2025-07-29|更新于2025-08-13
|浏览量:

RSA简介

1.encrypt

生成两个大素数p,q

计算 \[ N = pq \] 作为模数，计算Euler函数 \[ \phi(N) = (p-1)(q-1) \] 然后随机生成公钥e,满足 \[ gcd(e,\phi(N)) = 1 \] 当N，e公开作为公钥之后

任何人都可以用公钥加密 \[ c = m^e \mod N \]

2.decrypt

计算私钥 \[ d = e^{-1} \mod \phi(N) \] 解密 \[ m = c^d \mod N \]

只有拥有私钥d的人可以解密

但是，往往由于参数生成不当，RSA可以被破解

相关数论算法

1.Eculid算法

用一个递归很容易实现

时间复杂度是O(\(log^2_2x\))

2.Extend Eculid算法

也是递归

注意到 \[ ax + by =(x,y) \] 由于x可以由y和x%y表示，那么

只需求 \[ a'r+b'y=(x,y) \] 的解，注意到 \[ a'(x-qy)+b'y=(x,y) \] 那么 \[ a'x+(b'-qa')y=(x,y) \] 即 \[ a = a',b=b'-qa' \] 此算法可以直接由gmpy2的库调用，即gmpy2.gcdext

返回的是d,x,y的元组

公钥e相关攻击

1.small exponent attack

如果e太小，我们可以直接枚举k开根,m如果比较小的话，甚至k=0 \[ m = \sqrt[e]{c+kN} \]

考虑最严苛的情况，m=N-1

那么也就是 \[ (N-1)^e = c+kN \] 由于c是小于N的，那么 \[ k \approx \frac{(N-1)^e}{N} < N^{e-1} \] 一般情况来说，假设N为1024bit \[ k = \frac{m^e-c}{N} \approx \frac{m^e}{N}-1 \] 我们的枚举上限大概是2^32左右，而且32需要非常久的时间，在有限时间内，我们不妨假设上限是2的16次方

那这样的话，可以知道 \[ m^e \approx 2^{1040} \] 如果e=7的话，m最多148位，也就是18个字左右

但实际上18个字其实是144-152之间，经过实测18个字的话k基本都是0

模数N相关攻击

1.共模攻击

如果Alice一直不更换模数N，只更换e，那么加密同一条信息就可以被攻击 \[ \begin{align} c_1 = m^{e_1} \mod N \\ c_2 = m^{e_2} \mod N\\ \end{align} \] 我们想办法统一指数，这样我们就可以使用孙子定理（等式一边都是x）

考虑Bezout定理 \[ xe_1+ye_2 = gcd(e_1,e_2) \] 我们可以用extend Eculid算法来计算x,y

1.\(e_1,e_2\)互素

这是比较简单的情况，因为 \[ xe_1+ye_2 = 1 \] 那么直接有 \[ m = c_1^x c_2^y \mod N \]

2.\(e_1,e_2\)不互素

这样的话其实就是情况1加上small exponent attack

如果两个的公因数不大的话，是可以解决的

私钥d相关攻击

1.d泄露

如果我们知道d，当然所有消息都能求解了，不过，知道d可以分解N？这就很反常识了！

d的相关等式只有一个 \[ de = k\varphi(n)+1 \] 那理论上来说，直接分解de-1就可以求解了 \[ de -1 =k\varphi(n) \] 但问题是，k和phi可能都是合数，那这样它们的因数会混淆，无法得到结果

其实归根结底就是，我们知道phi的倍数

那么，phi又涉及到哪个等式呢？ \[ a^{\varphi(n)} \equiv 1 \mod n \] 那么 \[ a^{k\varphi(n)} \equiv 1 \mod n \] 这个式子有什么用呢，我们可以开方，不断的开方

假设 \[ k\varphi(n) = 2^s t \] 我们的主要想法还是去分解n，那么就要找到一个数，他和n的最大公因数是p或者q

那么这样的数怎么去构造？

我们随机选择和n互素的a，如果 \[ a^t \not\equiv 1 \mod n \] 那我们就找到了一个非平凡平方根，那么假设 \[ y_0 = a^t \] 不断平方，知道 \[ y_j \equiv 1 \mod n,y_{j-1} \not \equiv \pm1 \mod n \] 一旦找到这样的，那么 \[ n |y_j-1,n|(y_{j-1}-1)(y_{j-1}+1) \] 那么很明显两个因式都不是n的倍数，那么只有一种可能，他们分别是p，q的倍数

所以 \[ p = gcd(y_{j-1}-1,n) \]

2.d太小（Wiener's Atta）

如果 \[ q<p<2q,d<\frac{1}{3}N^{0.25} \] 那么是可以回复出d的

注意到 \[ de = k\phi(N)+1 \] 展开一下就是 \[ de = k(N-p-q+1)+1 \] 我们把公钥都放一起，就是已知的e和N，同时不妨记p+q=s

那么 \[ de = kN-ks+k+1 \] 即 \[ \frac{e}{N}=\frac{k}{d}-\frac{ks-k-1}{Nd} \] 注意到 \[ \frac{k}{d}-\frac{e}{N}=\frac{ks-k-1}{Nd} \] 由于 \[ 2q < p+q < 3q \] 而 \[ q^2 < N< 2q^2 \] 则 \[ 2\sqrt{N} < p+q < 3\sqrt{N} \] 由 \[ ed > k\phi(N),e<\phi(N) \] 则 \[ k < d \] 于是有 \[ |\frac{e}{N}-\frac{k}{d}|=\frac{ks-k-1}{Nd} < \frac{3d\sqrt{N}}{Nd}=\frac{3}{\sqrt{N}}<\frac{3}{9d^2}<\frac{1}{3d^2}<\frac{1}{2d^2} \] 根据Legendre定理

k/d是e/N的渐进分数，于是就求出了k和d
