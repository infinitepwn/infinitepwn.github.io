## 第一章 · 概论

### 1.1 计算机网络的定义与分类

**什么是计算机网络？**
- 一组互联的自治计算机的集合
- "互联"：用通信线路和设备连接
- "自治"：每台计算机独立工作，不受其他计算机控制

**按覆盖范围分类：**
1. **PAN（个人区域网）**：~1米，如蓝牙耳机连手机
2. **LAN（局域网）**：几百米~几千米，如学校机房、公司内网
3. **MAN（城域网）**：一个城市，如有线电视网络
4. **WAN（广域网）**：几百~几千公里，如互联网骨干网

**🌰 生活例子**：你在家连 WiFi（LAN）→ 流量走运营商 4G/5G 网（MAN/WAN）→ 到达百度服务器（WAN）。整个链路穿越了多种网络。

### 1.4 三种数据交换方式

| 方式 | 原理 | 优点 | 缺点 | 典型应用 |
|------|------|------|------|----------|
| **电路交换** | 通信前先建立专用物理通路，全程独占 | 时延小、无抖动 | 建立时间长、信道利用率低 | 电话系统 |
| **报文交换** | 把整个消息作为一个单位存储转发 | 不独占信道 | 大报文时延高、需大存储 | 早期电报 |
| **分组交换** | 把数据分成小包，独立路由 | 效率高、可靠性强 | 有排队时延、有开销 | 互联网 |

**🌰 分组交换比喻**：像快递公司——你的货物（数据）被拆成多个包裹，每个包裹贴上地址标签独立发出，途中可能走不同路线，最终在目的地重新组装。

### 1.5 计算机网络的性能指标

**时延的四个组成部分**：
1. **发送时延** = 数据帧长度 ÷ 信道带宽
2. **传播时延** = 信道长度 ÷ 传播速率（~2×10⁸ m/s）
3. **处理时延** = 路由器/主机分析数据包所需时间
4. **排队时延** = 在路由器队列中等待的时间

**🌰 例子**：你发一个 1MB 的文件（8Mb），通过 100Mb/s 的宽带，到北京服务器（距离 1000km）：
- 发送时延 = 8Mb ÷ 100Mb/s = 0.08s
- 传播时延 = 1000km ÷ (2×10⁸ m/s) ≈ 0.005s
- 排队+处理时延：平时 ~10ms，拥堵时可达 500ms

## 第二章 · 物理层

### 2.2 数据通信基础理论

**奈奎斯特定理 vs 香农定理**：

**奈奎斯特定理（理想无噪声信道）**：
- 最大数据率 = 2W·log₂N（bps）
- W = 信道带宽（Hz），N = 信号的离散级数

**香农定理（有噪声信道）**：
- 最大数据率 = W·log₂(1 + S/N)（bps）
- S/N = 信噪比

**🌰 例子**：家用电话线带宽约 3kHz，信噪比约 30dB（S/N = 1000）：
香农极限 = 3000 × log₂(1001) ≈ 3000 × 10 = 30,000 bps = 30 kbps

## 第三章 · 数据链路层

### 3.2 差错检测技术

**CRC（循环冗余检验）**：
- 把数据看作一个大的二进制多项式
- 除以一个约定的生成多项式
- 把**余数（FCS帧检验序列）**附在数据后面
- 接收方做同样的除法，余数为0则无错

**🌰 CRC 直觉理解**：把数据看成一个大数，除以一个"密码数字"，把余数写在包裹上。收件方做同样的除法，余数一样就说明数据没有改变。

> **重要区分**：CRC 只能**检测**错误，不能**纠正**错误。以太网发现 CRC 出错就直接丢弃帧，由上层（TCP）负责重传。

### 3.4 以太网（局域网核心）

**CSMA/CD（载波侦听多路访问/碰撞检测）工作流程**：
1. **载波侦听**：发送前先监听信道，如果有人在说就等待
2. **多路访问**：信道空闲时才开始发送
3. **碰撞检测**：发送过程中持续监听，发现碰撞立即停止并发强化信号
4. **随机退避**：等待随机时间后重试（二进制指数退避算法）

**🌰 CSMA/CD 类比课堂发言**：
- 想发言前先听有没有人说话（载波侦听）
- 没有人说话时再开口（多路访问）
- 说的过程中如果和别人撞上了，立刻停下来（碰撞检测）
- 随机等几秒再重新举手（随机退避）

> **💡 现代以太网已经不用 CSMA/CD 了！** 现代以太网全部是**全双工交换机**：每个端口独享一条点对点链路，根本不会碰撞。

## 第四章 · 网络层

### 4.1.1 网络层提供的两种服务

**虚电路服务 vs 数据报服务**：

| 对比项 | 虚电路服务 | 数据报服务 |
|--------|------------|------------|
| **思路来源** | 借鉴电话网络的设计思想 | 互联网的实际做法（TCP/IP） |
| **连接建立** | 通信前必须建立虚电路 | 不需要建立连接，即发即走 |
| **路径** | 同一连接的所有数据包走**同一条路径** | 每个数据包**独立路由**，可走不同路径 |
| **数据包顺序** | 到达顺序与发送顺序相同 | 可能乱序到达，需上层重排 |
| **可靠性保证** | 由**网络负责** | 由**端系统（TCP）负责**，网络尽力而为 |
| **典型代表** | ATM、帧中继、X.25 | 互联网 IP 协议 |

**🌰 虚电路服务类比**：像打电话——先拨号建立通话，通话期间所有声音都经过同一条线路，挂断后线路释放。

**🌰 数据报服务类比**：像寄快递——每个包裹贴上目的地址独立发出，快递公司根据当前状况决定走哪条路。

### 4.2.2 IP 地址

**IPv4 地址结构**：
- 32位二进制，通常用**点分十进制**表示，如 192.168.1.100
- IP 地址 = **网络号（Net ID）** + **主机号（Host ID）**
- 用子网掩码（Subnet Mask）来划分边界

**🌰 子网划分例子**：
给你 192.168.1.0/24 这个 C 类网络（254台主机）：
- 用 /25（255.255.255.128）划分成2个子网：
  - 子网1：192.168.1.0  ~ 192.168.1.127（128地址，126台主机）
  - 子网2：192.168.1.128 ~ 192.168.1.255（128地址，126台主机）

### 4.2.3 IP 地址与 MAC 地址

**两种地址的本质区别**：

| 对比维度 | IP 地址（网络层） | MAC 地址（数据链路层） |
|----------|-------------------|------------------------|
| **别称** | 逻辑地址 / 软件地址 | 物理地址 / 硬件地址 |
| **长度** | IPv4：32位 | 48位（6字节） |
| **分配方式** | 动态分配，可以改变 | 由网卡厂商固化在 ROM 中 |
| **作用范围** | 整个互联网 | 仅限同一局域网内 |

**🌰 通俗比喻**：
- **MAC 地址** = 你的**身份证号**（固定，出生就定，全球唯一）
- **IP 地址** = 你的**家庭住址**（随你搬家而变，方便定位）

**ARP（地址解析协议）**：
- 作用：根据已知的目标 IP 地址，去查询对应的 MAC 地址
- 工作过程：
  1. 主机A广播："谁是 192.168.1.20？"
  2. 主机B单播回应："我的 IP 是 192.168.1.20，我的 MAC 是 BB:BB:BB:BB:BB:BB"
  3. 主机A将映射存入 ARP 缓存（有效期约2分钟）

## 📝 全书核心要点速记

### 第一章 · 概论
- 三种交换：电路 / 报文 / **分组**交换
- 四个时延：发送 / 传播 / 处理 / 排队
- OSI 7层 vs TCP/IP 4层
- 协议=水平通信规则，服务=向上提供能力

### 第二章 · 物理层
- 奈奎斯特（无噪）vs 香农（有噪）
- 数字信号优于模拟（可再生）
- FDM / TDM / CDM / WDM 四种复用
- 光纤：全反射，单模远，多模近

### 第三章 · 数据链路层
- CRC 检错（不能纠错）
- PPP：广域网链路协议，字节填充
- CSMA/CD："先听后说，冲突退避"
- 交换机：MAC 自学习 + 泛洪转发

### 第四章 · 网络层
- 子网划分：IP AND 掩码 = 网络地址
- ARP：IP → MAC（广播请求）
- ICMP：ping (echo) / traceroute (TTL)
- RIP（跳数/距离向量）vs OSPF（链路状态）
- NAT：私有IP→公网IP共享上网
- IPv6：128位，固定首部，源端分片
