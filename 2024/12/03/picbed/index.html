图床搭建过程

一、为什么要搭建图床

起因在于，在本地编写markdown文件时，我们的图片都是保存在自己的电脑本地的，typora编辑器根据本地路径，寻找图片，但是，当我们把markdown上传到博客网站或者论坛时，本地的图片无法访问

因此需要使用网络路径，来上传我们的图片，这就需要用一个网站来存储图片

不过，

郑学长已经提供了他的图床，感谢前人栽树，因此本文接下来的内容没有什么意义

而且，github十分不稳定，上传的图片有时加载不出来

不过，由于我投入了时间搭建图床，出于沉没成本，我选择写一下搭建过程

二、用github(貌似Gitee也行？)仓库实现图床功能

1.注册github账号

首先，注册一个github账号

不过，国内github网站虽然可以访问，但十分不稳定，使用steam++(Watt Toolkit)，可以加快访问速度

2.创建一个仓库

注册完后

选择your repositories

new一个仓库

在repository name上，填写仓库名字，比如image，选择public,add a readme file

3、配置token

拖到下面选择developer settings

按照如图所示创建token

需要输入密码

note随便写，期限可以选择30天，但是等到期之后，需要重新修改

选择repo

然后就得到了token，中间那一大坨，记得保存，如果忘记了需要重新创建

接下来github上的操作就结束了

三、picgo上的配置

1、下载picgo

这个山大镜像站有

v2.3.1

选择64位的下载，安装之后

就来到了这个画面

2、配置github图床

选择图床设置，github图床

新建一个图床

图床配置名随便填

仓库名写自己的github账号名字/仓库名字

分支名，默认是main

token，将刚刚得到的token复制上去

存储路径，可以不写

自定义域名，这个就重要了，前面我就因为这个问题，无法上传图片，网上有的域名已经失效了

可以使用这个

https://cdn.jsdelivr.net/gh/账号名字/仓库名

其实就是gh/后面加上设定仓库名里填的

设置完之后

我们回到上传区

尝试拖拽一个图片上传，发现失败，原因是

开启了steam++！！

把加速器关掉

就会发现上传成功

这时候就可以在github上找到上传的图片了

至此，picgo也配置完了

但是，我们总不能一个一个上传，还需要配置typora

四、typora的配置

1、首先，下载typora

2、点击文件，偏好设置

选择图像栏目

上传服务设定选择picgoapp

picgo路径，选择自己安装的路径

配置完后，

点击验证图片上传选项

发现验证成功

然后，就配置完毕了

五、使用

当你写完markdown文件之后，选择格式

上传所有本地图片，然后，等待picgo上传完成

f072b7ffae0aecbbf3b4c03a125e6316](https://cdn.jsdelivr.net/gh/infinitepwn/blogimage/f072b7ffae0aecbbf3b4c03a125e6316.jpg

然后就发现图片变成了这种格式，接着

直接把markdown复制到论坛或者博客上，即可
